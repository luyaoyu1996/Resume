# 仓间中转调拨 - 物流中台核心履约能力

## 业务背景

跨境卖家在多仓履约场景下，货物需要在不同仓库之间中转调拨，典型场景包括：
- 国内三方仓 → 海外仓（如速卖通全托管备货、FBA 备货）
- 海外仓 → 平台仓（如美客多官方仓）

由于中转链路涉及多个独立系统（ERP + 三方仓 WMS + 平台仓），中间存在大量人工操作节点，效率低且容易出错。

---

## 两大场景对比

| 维度 | 场景A：国内三方仓→海外仓/平台备货仓 | 场景B：海外仓→平台仓 |
|------|------|------|
| 典型案例 | 发网→出口易、三方仓→FBA、三方仓→AE全托管备货仓 | 海外仓→美客多官方仓 |
| 产品标签来源 | ERP 通过接口自动获取，自动透传三方仓 | 客户从平台手动下载后上传 ERP |
| 箱唛来源 | ERP 通过接口自动获取，自动透传三方仓 | 客户从平台手动下载后上传 ERP |
| 装箱数据来源 | 三方仓完成装箱后回传 ERP | 原箱中转：客户在 ERP 维护后下发；拆箱换标：三方仓完成后回传 |
| 业务模式 | 拆箱换标（三方仓承担贴标重装） | 原箱中转 / 拆箱换标 两种模式 |

## 核心产品设计

### 1. 标签/箱唛流转（两种模式）

**接口自动化模式（场景A）：**
```
拉商品时自动调平台接口获取 SKU 标签 → 上传 COS 落库
  → 推出库单时自动透传三方仓
  → 入库单创建成功后自动请求箱唛
  → 箱唛落库 ERP → 自动透传三方仓
全链路零人工干预
```

**手动上传模式（场景B）：**
```
客户从平台下载商品标签/箱唛/预约信 → 上传 ERP
  → ERP 按附件类型分类存储（商品标签/箱唛/预约信三类）
  → 按业务阶段分批下发给海外仓
```

### 2. 装箱数据回传链路

**三方仓回传（场景A + 场景B拆箱换标）：**
```
三方仓完成装箱 → 回调 JAVA 接口
  → JAVA 转调 PHP
  → PHP 写入两张表：
      ship_batch_order_package（包裹级：长宽高重量）
      ship_batch_order_package_sku（SKU级：数量、编码）
  → 自动判断全量装箱完成 → 标记 is_pack_over=1
  → 前端展示【装箱完成】标签
```

**ERP 维护下发（场景B原箱中转）：**
```
客户在 ERP 维护装箱数据 → 下发给海外仓
```

### 4. 多阶段状态机控制（场景B）

| transfer_process_status | 含义 | 按钮状态 |
|------------------------|------|---------|
| 10 | 初始 | 正常可用 |
| 20 | 已下发换标任务 | 置灰，提示「等待海外仓装箱完毕」 |
| 30 | 装箱完成，可二次下发 | 高亮可用（补充箱唛） |

### 5. 附件分类传输（场景B）
- 商品标签、箱唛、预约信三类附件独立存储和传输
- 海外仓可精准识别附件类型，按需处理，解决原有系统无法区分导致的拒收问题

---

## 技术亮点

1. **虚拟仓三层路由判断**：`is_virtual` + `parent_warehouse_id` + `logisticsId` 三层判断，精确识别中转场景，控制触发范围不误伤其他链路
2. **状态机防重复下发**：`transfer_process_status` 控制按钮状态，避免重复下发换标任务
3. **装箱完成自动标记**：PHP 接收装箱数据后自动判断全量完成，无需人工确认
4. **报错边界设计**：「单据已下推三方仓」和「入库单未创建」两种拦截场景，保证数据一致性

---

## 商业价值

- 消除多个人工操作节点（创建入库单、获取标签、确认装箱、获取箱唛），大幅提升仓发效率
- 支持原箱中转和拆箱换标两种模式，覆盖不同客户的备货策略
- 附件分类传输解决仓库拒收问题
- 已对接三方仓：发网、出口易、易境通、马帮WMS 等；平台仓支持美客多、AE 官仓、FBA 等
