# 完美日记（逸仙）- 物流中台单据深度对接项目

## 项目背景

完美日记（逸仙电商）是马帮 ERP 的头部客户，因其保税仓业务复杂，要求 ERP 与海外仓之间的单据数据做**深度字段级对接**，而非标准接口。

该项目完成后，马帮收到 **140万定制费**。基于其扩展性设计，该套「单据深度对接框架」已向其他有同类需求的客户复用，成为物流中台单据扩展的核心基础能力。

## 接口设计详解：流程流转机制

### 整体架构：双端分层设计

```
┌─────────────────────────────────────────────────────────────┐
│                        马帮ERP层                             │
│  采购单 / 销售单 / 调拨单 / 手工单 / 加工单 / 退货单           │
└────────────────────────┬────────────────────────────────────┘
                         │ 单据下发（JAVA层）
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                  马帮镖局物流中台（JAVA）                      │
│  • 单据封装 & 下发（createHwcOrder / createHwcOrderApi）     │
│  • requestId 生成（base64编码的业务上下文，最长255位）          │
│  • 状态机驱动：wait → allow → finished / cancel / failed    │
└────────────────────────┬────────────────────────────────────┘
                         │ HTTP 推送 / 回调
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                   海外仓 WMS（第三方）                         │
│  • 接收入库单 / 出库单 / 退货单 / 调拨单 / 加工单              │
│  • 完成操作后，通过回调通知马帮                                │
│  • 回传：inboundSn / outboundSn、批次号、过期日期、箱号等      │
└────────────────────────┬────────────────────────────────────┘
                         │ 回调数据解析（PHP层）
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                   马帧PHP数据层                               │
│  • 解析回调字段，写入ERP单据（批次、箱号、数量差异等）           │
│  • 落库至 db_hwc_logistics_info（物流扩展信息表）              │
│  • 批次/箱号存储至专用批次管理表                               │
└─────────────────────────────────────────────────────────────┘
```

### 核心机制一：requestId 贯穿全链路

**设计动机**：多单据、多链路并发场景下，需要一个稳定的"主键"串联所有回调数据。

```
requestId = base64(
  production: "prd",
  companyId: 公司ID,
  tradeId: 单据ID,
  time: 当前时间戳
)

示例：eyJwcm9kdWN0aW9uIjoicHJkIiwiY29tcGFueUlkIjozNjA5OTcsInRyYWRlSWQiOjQwNTQsInRpbWUiOjE3MjQ4Mzk3NjZ9
```

- 马帮下发单据时生成并携带 `requestId`
- 海外仓所有回调（创建回调、发货回调、签收回调）均通过 `requestId` 反查原单据
- 解决了多次回调乱序、重复回调幂等等问题

---

### 核心机制二：双模式对接（异步推送 vs 同步查询）

**背景**：不同海外仓技术能力不同，有的支持回调，有的只能主动查询。

#### 模式A：异步推送模式（主流）

```
马帮 ──[1. POST 下发出库单]──► 海外仓WMS
                                   │
                               处理中...
                                   │
海外仓WMS ──[2. POST 创建回调]──► 马帮（/callback/create）
马帮 ──[3. ACK: 200 success]──► 海外仓WMS

海外仓WMS ──[4. POST 发货回调]──► 马帮（/callback/delivery）
马帮 ──[5. 回写运单号/批次]──► ERP
```

**关键字段**（创建出库单回调）：
```json
{
  "requestId": "...",    // 主键，反查原单据
  "status": "1",         // 1:创建成功 2:创建失败
  "outboundSn": "RD31470428116615"  // 海外仓系统单号
}
```

#### 模式B：同步直调模式（部分海外仓使用）

```
马帮 ──[POST /hwcOrder/createHwcOrderApi]──► 海外仓WMS
              ↑ AES-128-ECB 加密鉴权
海外仓WMS ──[即时返回 orderStatus]──► 马帮
```

**鉴权机制**：
```
Authorization = AES-128-ECB(
  data: "{companyId}##{loginUserId}##{timestamp}",
  secret: "协商密钥"
)

示例原文：100003##100025##1725443230
示例密文：+DDUkoteE6AyR7aK7yAauW5I9Q9fnIYYWTqcgim3/Oo=
```

### 核心机制三：单据状态机设计

#### 入库单状态机

```
下发成功
   │
   ▼
[wait: 等待入库]
   │
   ├──► [finished: 完成]       → 全量入库，触发库存更新
   ├──► [part: 部分完成]       → 实收数量 < 申请数量，差异预警
   ├──► [cancel: 取消]         → 单据作废
   └──► [failed: 失败]         → 入库失败，人工介入
```

**关键字段（入库签收回调）**：
```json
{
  "inboundStatus": "finished",
  "stockList": [
    {
      "applyQuantity": 23,      // 申请数量
      "deliveryQuantity": 23,   // 发货数量
      "receivedQuantity": 23,   // 实收数量（三者可能不一致）
      "erpSku": "BOS-0562-1",
      "produceCode": "BATCH-20240913001",  // ← 逸仙深度对接新增批次号
      "expireDate": "2025-06-03",          // ← 过期日期（保税仓必须字段）
      "productDate": "2025-06-01"          // ← 生产日期
    }
  ]
}
```

#### 出库单状态机

```
下发成功
   │
   ▼
[wait: 等待发货]
   │
   ├──► [allow: 已出单]        → 海外仓已受理，运单号已生成（必须携带trackNumber）
   │       │
   │       ▼
   │   [finished: 已发货]      → 货物已出仓，回传批次/箱号/运单号
   ├──► [cancel: 取消]         → 发货前取消
   └──► [failed: 失败]         → 发货失败
```

**关键字段（出库发货回调）**：
```json
{
  "orderStatus": "finished",
  "trackNumber": "9200190321831711230337",  // 运单号（allow状态必须有）
  "logisticsCode": "SF",                    // 物流公司编码
  "stockList": [
    {
      "applyQuantity": 5,
      "deliveryQuantity": 5,
      "produceCode": "BATCH-20240913001",   // ← 批次号
      "expireDate": "2025-06-03",
      "productDate": "2025-06-01",
      "packageNo": "1"                      // ← 箱号（逸仙深度对接扩展）
    }
  ]
}
```

### 核心机制五：批次/箱号全链路追踪（逸仙深度对接核心价值）

**痛点**：保税仓存在严格的海关监管，货物需要追踪批次、生产日期、过期日期。原标准接口不支持这些字段，导致盘点差异无法溯源。

**解决方案**：

```
步骤1：建表（#137387、#139397）
  ├── 新建 批次/箱号管理表（存储所有回传的批次数据）
  └── 新建 db_hwc_logistics_info（物流扩展信息表，落库所有扩展字段）

步骤2：接口扩展（PHP层）
  ├── 入库回调接口 新增 produceCode / expireDate / productDate 字段解析（#137510）
  ├── 出库回调接口 新增 produceCode / packageNo 字段解析（#137433）
  └── 调拨回调接口 新增批次数据（#137514）

步骤3：JAVA层下发/回调对接
  ├── 采购入库单下发 带上所有扩展字段（#137145）
  ├── 销售出库回调 全量批次箱号回传（#137505）
  ├── 退货出库回调（#137490）
  ├── 调拨入库/出库回调（#137509、#137512）
  └── 手工单/加工单回调（#137735、#137736）

步骤4：状态解析精细化（#139010）
  └── 入库回调新增 inboundStatus 状态机解析（finished/part/cancel/failed）
```

**字段贯穿路径**：
```
采购入库（带批次）
    ↓ 回调 produceCode + expireDate + productDate
[批次管理表]
    ↓ 调拨出库（带箱号 packageNo）
[调拨批次跟踪]
    ↓ 销售出库（回传 produceCode + packageNo）
[出库批次确认]
    ↓ 退货入库（批次匹配）
[批次核销闭环]
```

## 核心技术亮点

1. **批次/箱号全链路追踪**：从采购入库 → 调拨 → 销售出库 → 退货，全链路打通批次数据回传，解决保税仓盘点差异无法追溯的问题
2. **requestId 幂等设计**：base64编码业务上下文（companyId + tradeId + timestamp），所有回调均以此为主键，防止重复处理
3. **双模式兼容**：异步回调（WMS主动回调）+ 同步直调（马帮主动拉取），适配不同技术能力的海外仓
4. **双端分层解耦**：PHP 层负责数据解析回写 ERP，JAVA 层负责与海外仓通信，职责清晰
5. **sourceId 路由机制**：单套接口协议通过 sourceId 支持8类单据来源，避免为每种单据单独开发接口
6. **状态机完备**：入库（finished/part/cancel/failed）、出库（wait/allow/finished/cancel/failed）覆盖所有异常场景
7. **框架可复用**：基于逸仙项目抽象出「单据深度对接框架」，支持其他客户快速复用，降低定制开发成本 50%+

## 核心机制六：SAP 数据回传（逸仙企业级对接延伸）

### 背景

完美日记（逸仙）内部使用 SAP ERP 系统作为财务和供应链管控中枢。马帮ERP在完成出入库单据流水落库后，需将关键业务数据**回推至逸仙SAP**，实现双系统数据闭环。

### 需求编号

| 需求编号 | 名称 | 说明 |
|---------|------|------|
| #136749 | 逸仙海外项目 - SAP回调 | 整体SAP回传框架，和SAP联调时间2024年5月28日，上线6月11日 |
| #168309 | 2B销退入库单 - 回传SAP | 卢耀煜主导，2B销退场景下额外字段回传 |

### 流程位置：数据流第五层

```
海外仓WMS
   │ 回调
   ▼
马帮物流中台（JAVA/PHP 解析落库）
   │ 流水产生后
   ▼
马帮ERP（单据状态更新 + 批次数据落库）
   │ ← 这里是本章节的起点
   ▼
【SAP 回传】→ 逸仙SAP系统（财务/供应链核销）
```

### 触发时机

- **单据产生流水后**（入库签收完成 / 出库发货完成 / 退货入库完成）
- 马帮ERP主动将流水数据推送至逸仙SAP

### 关键回传字段（以 2B销退入库为例，#168309）

```json
{
  "line_sequence_number": "xxx",  // 订单维度行序号，取自 ship_batch_order.line_sequence_number
  "shipQty": 23,                  // 按照流水实际入库数量下发（非申请数量）
  "ShipQtyUOM": "EA"              // 单位，固定值 EA（Each）
}
```

**设计细节**：
- `shop_code`：早期版本包含，后经与客户确认去掉（见#168309描述中划线）
- `shipQty`：以**流水实际数量**为准，而非原计划数量，确保SAP财务凭证准确
- `ShipQtyUOM`：固定为 `EA`，对应保税仓库存管理单位

### 2C/2B 单据路由说明

- **2B订单**：默认 `other` 平台的均为2B订单
- **2C订单**：目前客户2C渠道仅 Shopify（后续可能扩展）
- SAP回传时，需根据2C/2B类型走不同的数据路径和字段映射

### 完整数据流（含SAP回传）

```
采购入库/销售出库/退货 等单据
   │
   ▼
海外仓WMS 完成操作 → 回调马帮
   │
   ▼
马帮PHP层：解析批次、箱号、状态 → 落库
   │
   ▼
马帮ERP：单据状态更新、批次可见
   │
   ▼
SAP回传触发：流水数据 + 行序号 + 实际数量 → 逸仙SAP核销
   │
   ▼
逸仙SAP：财务凭证生成、库存核销完成
```

## 简历提炼（多版本）

### 详细版（面试/详细简历）

> **主导完美日记（逸仙）海外仓保税仓单据深度对接项目（140万定制）**
>
> 负责全项目产品设计与需求管理，主导拆分 23 个子需求、设计 16 个海外仓接口（JAVA/PHP双端）。核心设计：
> ① 基于 requestId（base64业务上下文）实现全链路幂等回调，串联入库→出库→调拨→退货全链路；
> ② 设计 sourceId 路由机制，单套接口协议支持采购单、手工单、调拨单、加工单、退货单等6类业务单据；
> ③ 在标准接口基础上扩展批次号（produceCode）、箱号（packageNo）、过期日期、生产日期字段，打通保税仓全链路批次追踪，解决盘点差异无法溯源问题；
> ④ 设计完备的入库/出库状态机（各5个状态），覆盖部分入库、失败重试、取消等异常场景。
> 项目后抽象为「单据深度对接框架」向其他客户复用，降低同类定制开发成本 50%+。
