### 1、通知地址
当有新数据或订单状态变化时，会发送 (POST)请求到合作用户平台请求处理

合作用户在镖局物流平台填写通知地址 例如：
```
https://www.myapp.com/receive
```

### 2、镖局物流通知参数
- 通知内容（Json 格式 BASE64 加密）
- base64_encode(json_encode(xxx))
- 字段

| 字段名称 | 字段类型 | 默认值 | 必填 | 备注 |
| ------ | --------| ----- | ----| -----|
| notify | string |  | true |通知类型，可选参数如下<br>[orderChange,refreshToken,orderCancel]<br> 1、orderChange 订单更新触发通知，<br>当有待处理订单提交到镖局物流平时，触发此通知。<br>通知时返回以下参数orderInfo,为镖局物流订单号集合 <br>2、refreshToken 仅当角色为ERP服务商时回调，通过<br/>回调获取卖家在ERP最新的授权信息<br> 3、orderCancel 订单拦截触发通知，<br>当有平台取消订单提交到镖局物流时，触发此通知。<br>通知时返回以下参数orderInfo,为镖局物流订单号集合|
| timestamp | int |  | true |当前 Unix 时间戳,10 位整型数字 |
| sign | string |  | true |通知密钥|


### 3.1、orderChange 通知内容示例：

```json
{
	"notify": "orderChange",
	"orderInfo": {
		"codes": [
			"内部订单号",
			"内部订单号",
			...
			"内部订单号"
		],
		"count": "返回单号数"
	}
}
```

### 3.1.1、orderCancel 通知内容示例：

```json
{
	"notify": "orderCancel",
	"orderInfo": {
		"codes": [
			"内部订单号",
			"内部订单号",
			...
			"内部订单号"
		],
		"count": "返回单号数"
	}
}
```
### 3.2.1、refreshToken 通知内容示例：

```json
{
	"notify": "refreshToken",
	"type": "platform:平台店铺授权；logistics:直发物流授权",
	"sourceId": "ERP系统关联 物流系统/平台的唯一编号，通过此编号ERP系统可获取到对应的授权信息",
	"site": "订单所属站点/目的国"
}
```

### 3.2.2、refreshToken 通知响应示例：

```json
{
	"ErrorCode": "错误编码：9999:成功；其他值：失败",
	"Message": "错误描述",
	"Data": {
		"sourceId": "ERP系统关联 物流系统/平台的唯一编号",
		"accessToken": "访问令牌",
                "platformShopId":"平台店铺id"
	}
}
```

- sign 通知密钥
    sign = md5(notify=通知内容&timestamp=当前Unix 时间戳 apiKey)

:----------- | :-----------:
 [0,5}        |     2min    
 [5,10}        |     5min
 [10,20}        |     10min
