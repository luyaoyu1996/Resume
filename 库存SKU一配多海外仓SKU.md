# 库存 SKU 一配多海外仓 SKU

## 背景与痛点

跨境卖家在实际发货场景中，同一个本地库存 SKU 可能对应多个海外仓 SKU，典型场景：

1. **FBA 死链退仓**：亚马逊销售链接死链后，货物退回海外仓重新上架，此时仓库直接扫平台 SKU（如 FNSKU/X00 条码）操作入库，无需换标，节省换标费
2. **多店铺同品**：同一本地产品对应多个亚马逊店铺的 FNSKU，每个店铺一个海外仓 SKU
3. **PDA 扫码习惯**：部分海外仓 PDA 直接扫亚马逊 X00 条码上架，需要支持多 SKU 映射

## 核心产品设计

### 功能开关
- 企业级配置项：`overseas_stocksku_relate_multi_hwcproduct`（1:开启 / 2:关闭）
- 路径：仓库 → 海外仓服务商 → 产品列表 → 配对设置

### 出库优先级
- 一配多开启后，同一仓库下一个库存 SKU 对应多个海外仓 SKU 时，支持配置**出库优先级**
- 优先级字段 `outbound_priority` 控制出库时路由到哪个海外仓 SKU

### 退货入库路由逻辑（#104745）
退货时需要路由回正确的海外仓 SKU，避免仓库无法扫码入库：

```
开启一配多 + 一个库存SKU对应多个海外仓SKU时：
  → 获取所有候选海外仓SKU列表
  → 判断原出库仓 == 退货入库仓？
    → 是：取原出库时的 storageSku，与候选列表取交集
      → 有交集：直接用交集中的SKU
      → 无交集：按优先级取最高的
    → 否（跨仓退货）：直接按优先级最高的SKU路由
```

## 商业价值

- 节省换标费：退仓时仓库直接扫平台 SKU 入库，无需重新贴标
- 支持多店铺同品场景，提升海外仓商品管理灵活性
- 退货路由精准匹配原出库 SKU，避免仓库操作混乱
