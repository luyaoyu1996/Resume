# 马帮 AE 仓发对接项目（发网 → 出口易）

## 项目背景

**AE（速卖通）仓发**是马帮 ERP 物流中台的一条核心履约链路：卖家将货物从发网（国内仓）发往出口易（海外仓），再由出口易完成最终履约发货。

该项目的核心产品诉求是：
- 打通发网（出库）→ 出口易（入库）的调拨链路
- 实现**预装箱（装箱数据回传）**和**箱唛（Carton Mark）**的自动化流转
- 减少人工操作节点，提升仓发效率

## 核心业务流程

### 整体链路

```
卖家操作ERP
   │
   ▼
【出库单生成】（发网 → 出口易 调拨）
   │ 勾选"自动创建入库单"
   │
   ▼
【ERP自动生成对应入库单】（#64799、#64999）
   │
   ├──► [JAVA] 下发出库单到发网（携带海外仓SKU商品标签）（#64804、#64805）
   │
   └──► [JAVA] 下发入库单到出口易（#64688）
              │ 出口易入库单创建成功后
              ▼
         [JAVA] 自动请求出口易箱唛（#64743）
              │
              ▼
         [PHP] 接收箱唛URL，写入ERP（#64742）
              │
              ▼
         [PHP] 触发箱唛提交给JAVA（#64786）→ [JAVA] 转发给发网（#64779）
```

### 装箱数据回传链路

```
发网完成装箱操作
   │
   ▼
发网 ──回调──► [JAVA] 装箱数据回调接口（#64741）
                   │
                   ▼
              [PHP] 装箱数据接收接口（#64740）
                   │
                   ├── 写入 ship_batch_order_package（包裹维度）
                   ├── 写入 ship_batch_order_package_sku（SKU维度）
                   └── 判断所有商品是否装箱完成
                              │ 若全部完成
                              ▼
                         ship_batch_order.is_pack_over = 1
                         前端展示【装箱完成】标签（#64980）
```

## 关键接口设计

### 1. 出库单下发（ERP → 发网）

**接口**：`POST /hwcOrder/createHwcOrderApi`  
**鉴权**：AES-128-ECB（companyId##loginUserId##timestamp）  
**sourceId**：15（速卖通仓发专用）

**SKU标签字段**（#64804新增）：
```json
"item": [
  {
    "erpSku": "BOS-0562-1",
    "commodityPDF": "HTTP:xxx.PDF",  // 商品贴（单个PDF）
    "goodsLabelConfigInfo": {}        // 商品标签配置信息（2025-01新增）
  }
]
```

**SKU标签来源**（#64705、#64805）：
```
① PHP 拉商品/盘点时，调出口易标签接口 → 上传COS → 存 db_hwcproduct.cos_hwcsku_label
② 判断：调拨单起始仓=发网（id=2615），目的仓=出口易（id=60）
③ 满足条件则提交 cos_hwcsku_label 给JAVA出库单下发接口
④ 支持虚拟仓判断：db_warehouse.is_virtual=1 → 找父仓 → 找 logisticsId
```

### 2. 入库单下发（ERP → 出口易）

**接口**：出口易 OpenAPI  
**对接接口**（#64688）：
- `POST /v1/inboundOrders` — 创建入库单
- `GET /v1/inboundOrders/{packageId}/cancel` — 取消入库单
- `GET /v1/inboundOrders/{shipmentId}/status` — 查询入库单状态

### 3. 装箱数据回传

**发网 → JAVA 回调**（#64741）：复用 FBA/AliExpress 已有的装箱回调接口结构

**JAVA → PHP 转调**，PHP 写入两张表：
- `ship_batch_order_package`（包裹级数据：长宽高重量、包裹号）
- `ship_batch_order_package_sku`（SKU级数据：数量、SKU编码、平台SKU）

**报错场景设计**：
- 入库单已下推三方仓：返回 `"单据已下推三方仓，无法写入装箱数据到erp"`
- 入库单未创建：返回 `"入库单未创建，无法回写装箱数据"`

### 4. 箱唛（Carton Mark）自动化

**触发时机**：出口易入库单创建成功后，自动触发

**全链路**：
```
出口易入库单创建成功
   │
   ▼（#64743）
JAVA 调出口易箱唛接口：GET /v1/inboundOrders/{ShipmentId}/label
   │
   ▼（#64742）
PHP 接收箱唛接口：入参(requestId, order_no, print_url)
→ 落库 ship_batch_order.box_mark_file_path
   │
   ▼（#64786）
PHP 调 JAVA 箱唛下发接口（携带箱唛 PDF URL）
   │
   ▼（#64779）
JAVA 调发网箱唛提交接口
→ 发网贴箱唛，准备出库
```

## 核心设计亮点

1. **出入库单联动创建**（#64799、#64999）：操作出库单时，勾选后自动关联创建入库单，避免人工二次创建，支持企业维度勾选记忆，调拨单已存在入库单时按钮置灰防重复
2. **SKU商品标签自动化**（#64705、#64804、#64805）：拉商品时自动获取出口易SKU标签并上传COS，推单时自动透传，发网仓库人员可直接打印贴标，无需人工获取
3. **虚拟仓路由判断**：通过 `db_warehouse.is_virtual` + `parent_warehouse_id` + `logisticsId` 三层判断，自动识别发网→出口易的调拨场景，精确控制触发范围
4. **装箱完成自动标记**（#64740）：PHP 接收装箱数据后自动判断是否全量完成，自动设置 `is_pack_over=1`，前端展示标签，客户无需手动进入装箱页面确认
5. **箱唛全自动流转**（#64742、#64743、#64779、#64786）：从出口易获取箱唛→落库ERP→透传发网，全链路自动化，无人工干预节点
6. **报错边界设计完备**：装箱数据写入设计了"单据已下推"和"入库单未创建"两种拦截场景，保证数据一致性
