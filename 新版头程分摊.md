# 新版头程分摊 - 物流中台核心能力升级

## 背景

跨境电商海外仓业务中，头程费用（运费、关税、杂费等）需要按 SKU 分摊到每笔出库订单，影响利润核算准确性。旧版分摊逻辑存在以下问题：
- 只支持单程分摊，无法处理多段头程（如转仓、多次补货）
- 初始化不友好，历史数据迁移困难
- 费用变更后无法重新分摊，数据不可修正
- 分摊粒度粗，无法基于具体出库单/入库单做精细化分摊

## 四大核心特性（结合代码与设计文档）

### 1. 支持多程分摊
- 三表设计：批次主表（`db_head_shipping_fee_ship_batch`）+ 明细表（`db_head_shipping_fee_ship_batch_detail`）+ 分摊记录表（`db_head_shipping_fee_ship_batch_record`）
- 每次头程对应一个独立批次，同一 SKU 可关联多个批次，按 FIFO 先进先出策略消费
- 分摊时按批次维度独立计算，互不干扰，支持跨批次分摊直到满足出库数量
- 代码体现：`InboundHeadShippingFeeComponent.saveExcelHeadShippingFeeShipBatch()` 按 `batchOrderId` 分组，每个入库单独立生成批次

### 2. 初始化友好
- 支持 Excel 导入初始化，兼容历史数据迁移
- 支持「追加初始化」：已有批次可追加新 SKU，无需重建（判断 `headId` 是否存在走不同路径）
- 支持总价/单价两种录入模式（`feeType=1` 总价系数=总数；`feeType=2` 单价系数=1）
- 支持组合商品分摊：通过 `comboProductDto2Stock()` 将费用按比例分解到组成 SKU

### 3. 支持变更费用后重新分摊
- 费用变更后，系统检测批次分摊状态（`SHARED=2` / `PARTIAL_SHARE=1`）
- 自动触发撤销旧分摊记录（`revokeHeadShippingFeeShipBatchNew`），将记录标记 `detail_type=2`，恢复未分摊金额
- 删除旧明细后重新写入，保证数据一致性，全程有操作日志追溯

### 4. 基于出库单与入库单双维度分摊
- **入库单维度**：`InboundHeadShippingFeeComponent` 处理，绑定 `batchOrderId`、`erpInOutOrderNo`，初始化批次费用
- **出库单维度**：定时任务扫描未分摊出库单 → `shareHeadShippingFeeShipBatchBySku()` 按 SKU 匹配可用批次 → `shareDetailByNum()` 按数量计算分摊金额 → 写入 `Db_OrderExtend` → 触发利润重算（`Db_NeedToCalculate`）
- 分摊结果精确到订单行项目，直接驱动 ERP 利润核算

## 完整业务流程

```
定时任务启动
  → 同步马帮ERP已发货订单（分页查询，避免OOM）
  → 转换为新版出库单保存
  → 头程分摊任务调度（按企业并行，按仓库串行）
    → 查询未分摊出库单
    → 按SKU匹配可用入库批次（FIFO）
    → 按数量分摊：运费 + 关税 + 其他费用
    → 生成分摊记录，更新批次状态
  → 汇总分摊结果
  → 同步回马帮ERP订单（写入头程费用，触发利润重算）
```

## 技术亮点

1. **分布式锁**：Redisson 锁键 `SHARE_HEAD_SHIPPING_FEE_LOCK_{companyType}_{companyId}_{warehouseId}`，同仓库串行执行，防并发重复分摊
2. **精度控制**：所有金额计算统一 `SCALE=8`，`RoundingMode.HALF_UP`，8位小数精度
3. **多币种支持**：费用录入支持原币，通过汇率（`currencyRate`）统一换算为 RMB 后分摊
4. **费用类型扩展**：运费、关税之外支持动态扩展其他费用类型（`other_fee_detail_json` JSON字段），无需改表
5. **企业级灰度开关**：`HEAD_SHIPPING_FEE_SHIP_BATCH_ENABLE` 配置项控制企业是否启用新版，平滑迁移
6. **数据自愈**：`verifyAndFixShareNumConsistency()` 定期校验并自动修复分摊数量不一致问题
