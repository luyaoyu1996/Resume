## 七、4 个高风险追问 + 标准应对

### 🔴 风险 1：业务跨度（物流 → 选品/Listing/美工）

**追问**：
> "我们要做的是选品、Listing、产品图、供应商管理的 AI，你之前做的是物流履约，跨度会不会有点大？"

**标准应对**：

> "我是**在跨境电商供应链大盘下**做物流和海外仓的，所以对选品上架、库存铺货、供应链打通这些上游环节每天都在对接。举两个例子——
>
> 一是 **OpenClaw 智能采购员**，核心场景就是**采购员日均 400+ 单的异常处理**，这其实就是**供应商管理和采购下单的 AI 化**。做这个项目我**现场跟采购员跟单了两周**，对"跟单 / 找替代 / 降本 / 采购量计算"四大流程理解非常深。
>
> 二是**头部客户定制**，对接的客户从美妆到 3C 全品类都有，**选品节奏、Listing 运营节奏我都很熟悉**。
>
> 所以选品 / Listing / 美工这些场景我不敢说"做过"，但**业务逻辑我熟、上手不会慢**。"

---

### 🔴 风险 2：AI 类型定位（B端SaaS AI → 电商运营AI）

**追问**：
> "你做的 AI 产品都是内部提效工具，我们要做的是对接运营 / 采购 / 美工的业务 AI，场景不一样吧？"

**标准应对**：

> "我做的 AI 产品**本来就是面向业务一线岗位**的——
>
> **OpenClaw 是给采购员用的**：采购员原来日均 400+ 单要逐一检查跟单，AI Agent 做异常判断，他们完全不用懂技术。
>
> **Echo 是给 PM / 客服 / 开发用的**：30 位同事日常用，**包括 12 位 PM 深度用户 + 客服用 RAG 查业务逻辑**。
>
> 这两款都是典型的 **"非技术人员也能用的提效 AI"**，和贵司 JD "降低非技术人员使用门槛" 完全一致。**AI 产品的难点不在模型，在场景抽象和工作流设计**——这一点我是真走过场的。"

---

### 🔴 风险 3：工具链差异（Dify vs n8n/coze/影刀）

**追问**：
> "我们用的是 n8n / coze / 影刀，你之前用的是 Dify，这些你上手快吗？"

**标准应对**：

> "**Dify 和 n8n / coze 底层逻辑相通**——都是节点式拖拉拽 + 变量传参 + LLM 调用 + 外部 API 集成。
>
> 我用 Dify 做 Echo 时**深度做过工作流编排**（PRD 生成、UI 生成、视频脚本等多个 workflow），**迁移到 n8n 或 coze 不会有本质障碍**。
>
> **影刀 RPA 我了解原理但没深度用**，我可以在入职前自学。我的 Python 底子和 AI Coding 能力足够支撑快速上手。"

---

### 🔴 风险 4：业务认知考察（AI 选品 / AI 生图 / AI Listing）

**追问示例**：
- "你怎么看 AI 选品？"
- "Listing 文案用 AI 生成，你觉得最大的坑是什么？"
- "AI 批量生图在亚马逊，主图二图有什么差异？"

**核心应对锚点**：

| 场景 | 核心观点 |
|------|---------|
| **AI 选品** | 数据驱动（销量/趋势挖爆品）+ LLM 做差异化 + **必须前置规避侵权** |
| **AI Listing** | 三大坑：**关键词堆砌被降权 / 品牌调性不一致 / 平台禁用词未过滤**。解法：**品牌调性 + 关键词库 + 合规词库三模板叠加** |
| **AI 生图** | **主图严格白底合规，二图可场景化**；批量生图最大的坑是**"AI 感太重"**，解法是**prompt 模板沉淀 + LoRA 保品牌一致性** |
| **AI Agent 做业务自动化** | **拆解成"AI 可执行最小单元"，每一步 AI 做判断、CLI 做执行** |
| **B 端 AI 落地原则** | **必须有"人工 review 关卡"和"一键回滚"** — 一次批量改价错误就是几十万损失 |

## 八、跨境业务 7 环节全景陈述（2 分钟硬货 · 背这个）

> "我想先整体讲讲我对跨境电商业务的理解，再落到 AI 能在哪些环节创造价值。
>
> **跨境电商主流程可以拆成 7 个环节，每个环节都有清晰的业务痛点和 AI 机会：**
>
> **第一是选品和开发**。卖家每天要从海量市场数据里挑爆品——看 BestSeller 榜、Helium 10 / Jungle Scout 的销量趋势、竞品 Review、价格带分布，**选出"有需求缺口、竞争不激烈、利润可接受"的品**。选完之后是产品开发——找供应商打样、差异化设计、算打样成本、测 MOQ。
>
> **第二是采购和备货**。上架前算**首单备货量**，上架后按销售节奏**补货**。痛点是**备货节奏失控**——备少断链、备多积压；头程海运 30-45 天、空运 5-7 天，**时效和成本每次都要权衡**；还有 1688 采购端的**找替代、催货、议价**，日均几百单的采购员是纯手工作业。
>
> **第三是 Listing 撰写与上架**。Listing 是卖家在亚马逊 / Shopee / TikTok 的"门面"——**五点描述字数控制、关键词埋词密度、长短尾词组合、品牌调性一致性**，写一篇优秀 Listing 平均 2-4 小时，**还要规避平台合规词、侵权词**。批量上架场景下，500 SKU 是纯苦力。
>
> **第四是美工和视觉资产**。**主图纯白底突出产品、二图场景化卖点可视化、A+ 页面做品牌故事**。美工还要出**广告创意图、短视频素材、旗舰店装修图**。一个 SKU 完整视觉资产要 3-5 天。
>
> **第五是广告和运营**。SP/SB/SD 广告投放、**关键词出价、ACOS 控制、活动报名（秒杀/会员日/黑五网一）**、价格调整、评价管理。日常运营动作非常碎，靠人工盯。
>
> **第六是履约和物流**。**FBA 头程发货、海外仓备货、Y2 货代转运、面单打印、物流拦截、费用测算、退换货处理**——这是我过去两年主战场，连接 1800+ 物流商和 1000+ 海外仓。
>
> **第七是售后和复盘**。差评处理、退款申诉、**客户 Review 反哺选品**、月度财务对账、利润核算。
>
> **AI 在这 7 个环节切入点完全不一样**——选品是数据分析 + LLM 差异化；Listing / 美工是 AI 生成；采购是 AI Agent 做决策自动化；运营是 RPA 批量操作；履约是智能测算和异常预警。**贵司 JD 提到的选品、Listing、产品图、供应商管理，正好覆盖前 4 个环节，也是 AI 落地 ROI 最高的区段。**"

## 八补 · 专业术语对照表（先理解再背）

> 按 7 环节顺序列出，每个术语**一句话说清是什么、卖家用来干嘛**。

### ① 选品 / 开发

| 术语 | 通俗解释 |
|------|---------|
| **BestSeller 榜** | 亚马逊各品类的**销量榜 Top 100**，选品第一入口 |
| **Helium 10 / Jungle Scout** | 亚马逊**第三方选品工具**，看 SKU 销量、关键词搜索量、竞品数据 |
| **竞品 Review** | 竞品详情页里的**客户评价**，差评能反推产品改进点 |
| **MOQ** | Minimum Order Quantity，**最小起订量**。供应商要求你"至少订 500 件我才给你生产" |
| **打样** | 供应商按你的设计图**先做 1-2 件样品**给你确认，确认后再量产 |

### ② 采购 / 备货

| 术语 | 通俗解释 |
|------|---------|
| **头程** | 货物从**国内仓到海外仓的第一段运输**（海运 30-45 天 / 空运 5-7 天） |
| **尾程** | 海外仓**到买家手上**的最后一段（USPS/FedEx 等本地承运商送货上门） |
| **备货 / 补货** | 备货 = 上架前首次进货；补货 = 持续销售中根据库存节奏再采购 |
| **断链** | 库存卖完还没到货，**商品被迫下架或停售**（最怕的情况，链接权重会掉） |
| **积压** | 备太多卖不动，**占仓储费 + 压资金** |
| **1688** | **阿里巴巴 B2B 批发平台**，跨境卖家最主要的国内供应商来源 |
| **找替代** | 现有供应商停产/涨价/质量问题时，**找等效的新供应商**替换 |

### ③ Listing 撰写与上架（⚠️ 这是面试重点）

| 术语 | 通俗解释 |
|------|---------|
| **Listing** | 卖家在平台上的**一个完整商品详情页**（标题+五点+图片+描述+A+） |
| **刊登** | 中文 SaaS 通用叫法，意思**约等于 Listing**（侧重"发布上架"这个动作） |
| **五点描述 / Bullet Points** | 亚马逊 Listing 标题下方的 **5 条卖点短描述**，每条 <200 字符，是买家第二个看的地方 |
| **关键词埋词 / 长短尾词** | 卖家主动**把用户搜索词嵌入 Listing 文案**，帮助被搜到；"长尾词"= 具体少人用（"男士跑步保暖压缩裤"），"短尾词"= 泛但高竞争（"跑步裤"） |
| **SKU** | Stock Keeping Unit，**库存最小单元**，同一产品的不同颜色/尺寸算不同 SKU |
| **品牌调性** | 品牌整体语气风格（专业/活泼/高端/性价比...），**多 SKU 要保持一致** |
| **侵权词 / 合规词** | 亚马逊**禁用词**列表（如"最便宜"、"100% safe"、"cure cancer"），出现会被下架 |

### ④ 美工 / 视觉资产（⚠️ 面试可能追问）

| 术语 | 通俗解释 |
|------|---------|
| **主图** | Listing **第一张图**，必须**纯白底 + 商品占 85%+**，是亚马逊硬性规定 |
| **二图-七图** | 主图之后的**场景图、细节图、使用图、对比图、尺寸图**等，可以自由创作 |
| **A+ 页面** | **亚马逊品牌注册后才能用的富文本图文描述页**，在 Listing 下半部分，大量图+文讲品牌故事和产品细节（A = Amazon 升级版内容） |
| **EBC** | Enhanced Brand Content，**A+ 页面的旧称**，Shopee / eBay 等平台也有类似的"富文本商品描述" |
| **旗舰店装修** | 品牌在平台的**独立店铺首页**（Brand Store），类似淘宝天猫旗舰店，要视觉统一 |
| **LoRA** | **小型模型微调技术**，用 50-200 张品牌素材训练一个小模型，AI 生图时**保持品牌统一风格** |

### ⑤ 广告 / 运营（⚠️ 面试可能追问）

| 术语 | 通俗解释 |
|------|---------|
| **SP 广告** | Sponsored Products，**亚马逊商品推广广告**，最基础最常用的广告类型 |
| **SB 广告** | Sponsored Brands，**品牌推广广告**，展示品牌 LOGO + 3 个产品，搜索页顶部 |
| **SD 广告** | Sponsored Display，**展示型广告**，可以投放到竞品详情页（"追击竞品"） |
| **ACOS** | Advertising Cost of Sales = 广告花费 / 广告销售额，**越低越好**（通常 20-30% 以下算健康） |
| **关键词出价** | 卖家为某个搜索词**设置愿意付的广告单次点击费**（CPC） |
| **秒杀 / LD** | Lightning Deal，亚马逊**限时闪购活动**，流量大但利润薄 |
| **会员日 / Prime Day** | **亚马逊 Prime 会员专属大促**，类似双十一 |
| **黑五网一 / BFCM** | Black Friday + Cyber Monday，**感恩节后最大购物节**，跨境卖家全年最重要的旺季 |
| **ASIN** | Amazon Standard Identification Number，**亚马逊上每个 SKU 的唯一 ID** |

### ⑥ 履约 / 物流（自己主战场）

| 术语 | 通俗解释 |
|------|---------|
| **FBA** | Fulfillment by Amazon，**亚马逊自营履约**——货先发到亚马逊仓库，由亚马逊打包发货 |
| **FBM** | Fulfillment by Merchant，**卖家自发货**（通过自己的海外仓或直发） |
| **海外仓** | 卖家**在海外国家租的仓库**（不是亚马逊仓），灵活但自己要管理 |
| **Y2 货代** | 货代模式：**货代先揽收国内仓→飞海外仓→等平台面单→再发买家手上**（本次项目 #87541 的背景） |
| **面单** | 包裹上**贴的运输标签**，印有收货人地址、承运商编码、追踪号 |
| **物流拦截** | 客户取消订单后，**通知承运商不要发出**（哪怕已经揽收） |
| **费用测算** | 下单前**自动算出各个物流渠道的运费、时效、承运商**，选最优 |
| **仓配分离** | 一个订单**仓库负责打包、承运商另外揽件**，不是仓库直接发。需要提前告知仓库揽件方便分堆 |
| **预报 / 预报单** | 卖家**提前告诉仓库有多少货要来 / 要发**，仓库好准备 |
| **揽收 / 揽件** | 承运商来仓库**取件**（UPS/FedEx 等上门取货） |

### ⑦ 售后 / 复盘

| 术语 | 通俗解释 |
|------|---------|
| **Review** | 客户在 Listing 下的**评价**（1-5 星 + 文字评论），亚马逊最重要的信任信号 |
| **差评申诉** | 遇到恶意 / 错误差评，**向亚马逊申请移除** |
| **利润核算** | 把**订单收入 - 物流 - 广告 - 平台佣金 - 仓储 - 退款**后的真实利润算出来 |
| **平台佣金** | 亚马逊收取的**销售分成**（通常 8-15%） |

## 八补·技术术语快查（AI / 工作流类）

这类是 JD 里涉及的**技术栈**，面试官可能追问——

| 术语 | 通俗解释 |
|------|---------|
| **LLM** | Large Language Model，**大语言模型**，如 GPT / Claude / 通义千问 |
| **AI Agent** | **会"思考 + 调用工具"的 AI**，不是简单问答，能根据任务自己决策调哪个工具 |
| **RAG** | Retrieval-Augmented Generation，**检索增强生成**，把公司内部知识库喂给 LLM，让它基于你的数据回答 |
| **工作流编排** | 把多个步骤（AI 生成 → 人工审核 → 入库）**串成自动化流水线**的工具（n8n / coze / Dify） |
| **RPA** | Robotic Process Automation，**机器人流程自动化**，模拟人在浏览器里点击/填表/复制的工具（影刀、UiPath） |
| **FastAPI** | **Python 写 Web API 的快速框架**，OpenClaw 后端用的就是这个 |
| **Webhook** | **平台主动推送消息给你系统的机制**（对比"你去问平台"叫轮询 / polling） |
| **n8n** | 开源的**工作流编排工具**，节点式拖拉拽，国外卖家团队常用 |
| **coze** | **字节出品的工作流编排平台**，主打 Bot 开发，国内团队常用 |
| **Dify** | **开源工作流编排 + RAG 一体化平台**，你做 Echo 用的就是这个 |
| **影刀** | **国产 RPA 工具**，跨境电商卖家批量操作平台常用 |
| **Plasmo** | **浏览器扩展开发框架**，你做 Echo 用的就是这个（基于 React/Vue 写 Chrome 扩展） |

## 八补·背诵 Tips

**背的顺序**：

1. **先看"八补"术语表** → 把每个词在心里过一遍，确保**都懂**
2. **再背第八节全景陈述** → 这时候每个术语你都有**具象理解**，不会磕巴
3. **最后练 30 秒快讲版** → 只保留 7 环节骨架 + 每环节 1 句核心痛点（面试时如果时间紧可以切换）

**遇到不确定的术语现场怎么办**：

- 千万**别硬用专业词但讲错**，**"不装懂"比"装错"好得多**

## 九、跨境 7 环节 AI 赋能深度分析（面试高阶话术）

> **核心心智模型**：AI 擅长 ≠ AI 应该做
> AI 擅长的：规则重复、数据密集、文本/图像生成
> AI 不擅长的：创造性、高风险决策、品牌调性把控
> **面试核心判断**：**每个环节要讲清 "AI 能做的" vs "人必须做的" 的边界**

---

### 9-0 AI 赋能价值矩阵（面试总览）

**ROI 极高 + 落地成熟**（可以当"应该首批做"来回答）：
- Listing 文案批量生成（LLM）
- Listing 翻译多语言（LLM）
- bid 调整（规则/算法/Agent）
- 采购催货跟单（Agent）
- 商品详情图批量生成（Midjourney/SD）

**ROI 高 + 技术挑战大**（可以当"二期规划"来讲）：
- AI 选品决策辅助（LLM + 数据分析）
- 差评 Review 情感分析反哺选品
- 智能补货预测（时序模型）

**风险高 / 不建议首批**（可以当"坑点警示"来讲）：
- 主图生成（高合规要求）
- 全自动改价（风险失控）
- AI 直接下单采购（风险失控）

---

### 9-1 ① 选品 / 开发

**AI 擅长点**：海量数据筛选 / 竞品 Review 分析 / 价格带利润模拟 / 趋势识别

**3 大坑点 + 解法**：
1. **AI 不能替代产品直觉** → AI 做候选池,人做最终决策
2. **AI 推荐易触发侵权** → 强制集成商标库+专利库前置检查(USPTO API)
3. **数据源有限有盲区** → 多站点数据源接入+差异化分析 Prompt

**面试话术**：
> "选品这块 AI 最大的价值是**替代数据筛选苦力,不是替代产品决策**。我会把 AI 定位成'选品助理',从 10 万 SKU 缩到 500 候选;**然后人来做最终决策**——产品直觉、市场节奏、品牌战略这些 AI 学不会。**最大的坑是侵权**——AI 输出的差异化设计可能踩专利,所以任何 AI 选品产品都要强制集成商标/专利库做前置检查,没这层兜底不要上线。"

---

### 9-2 ② 采购 / 供应商管理（你的主场,重点讲)

**AI 擅长点**：催货跟单 / 找替代供应商 / 议价话术生成 / 采购量计算 / 异常审核

**3 大坑点 + 解法**：
1. **1688 供应商数据不标准** → 工商数据+历史交易数据交叉验证
2. **全自动下单风险极高** → 大金额强制人工 Review(OpenClaw 里 >$500 必须确认)
3. **议价话术不能照搬** → AI 生成初稿 + 采购员个性化润色 + 人发送

**面试话术**:
> "采购这块正是我做 OpenClaw 的主战场。核心设计是'**AI 做异常判断、CLI 只取写数据**'——AI 负责'这单要不要催'、'是不是换供应商'这种判断,CLI 负责调接口执行。
>
> **关键坑是全自动下单风险极高**——AI 看错数据下单 10 万件是可以上公司头条的事故。所以 OpenClaw 里大金额订单强制人工 Review,**AI 只做决策建议,不做直接执行**——这是 B 端 AI 产品和 C 端玩具的根本区别。"

---

### 9-3 ③ Listing 撰写(面试最热)

**AI 擅长点**:标题+五点批量生成 / 多语言翻译 / 关键词埋词 / 合规词过滤 / A+ 页面图文

**4 大坑点 + 解法**:
1. **关键词堆砌触发 A10 降权** → Prompt 约束密度(每100字最多3个词)+自然度评分
2. **品牌调性不一致** → 沉淀品牌调性模板(如"专业70%+温暖30%")注入 Prompt
3. **侵权词/合规词漏检** → 独立合规词库+规则过滤(不能只靠 LLM 自检)
4. **五点描述字数超限** → Prompt 硬约束+代码后校验

**面试话术**:
> "Listing 是 AI 生成最成熟的场景,但坑也最多。我会分三层防护——
>
> **第一层 Prompt 工程**:品牌调性模板 + 关键词密度约束 + 字符数硬限制
>
> **第二层合规词库**:独立维护的亚马逊禁用词库,生成后强制过滤——**不能完全信 LLM 自检**,因为 LLM 训练数据滞后,平台规则在变
>
> **第三层人工 spot check**:批量生成 500 条 Listing,抽检 10-20 条送审,防止批量漏洞
>
> 最大的单点坑是**关键词堆砌**——AI 喜欢把关键词全塞进去,结果触发 A10 降权算法,反而比没关键词更惨。"

---

### 9-4 ④ 美工 / 视觉资产(面试高频)

**AI 擅长点**:二图场景图批量 / A+ 配图 / 广告创意图 / 短视频素材 / 主图背景替换

**4 大坑点 + 解法**:
1. **主图合规严,AI 生图易踩雷** → 主图不用纯 AI 生成,只做抠图换背景调光
2. **品牌一致性难保证** → 训练品牌 LoRA(50-200张素材微调)
3. **文字生成易错** → AI 生图 + PS/Adobe Firefly API 后处理加文字
4. **版权风险** → 用商业授权模型(Adobe Firefly)不用开源模型

**面试话术**:
> "美工这块最重要的认知是**不是所有图都适合 AI 做**。我会分三档——
>
> **主图**:**绝对不用纯 AI**,亚马逊主图合规极严,AI 的'虚假质感'被消费者一眼识破;AI 只做抠图、换背景、调光
>
> **二图/场景图/A+**:**AI 主战场**,容错度高可以大量产出
>
> **广告创意**:**AI 批量+A/B 测试**,日常跑 20 个版本选最优
>
> 关键是**品牌一致性**——Midjourney 裸生成每张风格不同,旗舰店会很乱。要训练**品牌 LoRA**保证风格统一。另外**AI 生文字永远会错**,文字用 PS 或 Adobe Firefly API 后处理。"

---

### 9-5 ⑤ 广告 / 运营(面试会深挖)

**AI 擅长点**:**bid 调整** / 关键词挖掘 / 广告活动复盘 / 活动排期 / Review 情感分析

**bid 调整 = 广告每次点击愿付最高价根据效果自动调整**(详见本节补充速记卡)

**3 大坑点 + 解法**:
1. **全自动改价风险极高** → 强制价格下限(成本+15%毛利)
2. **bid 调整过激** → 单次幅度±20%封顶 + 每日调整次数上限
3. **Review 情感分析过度解读** → 看统计显著性,10%以上客户反馈才采信

**面试话术**:
> "广告运营 AI 最成熟,但坑也最致命——**任何自动化动作都要有刹车**。
>
> **bid 调整**核心是'**单次幅度±20% + 每日次数上限 + 人工监控 dashboard**'
>
> **全自动改价我不建议做**——凌晨 3 点没人盯,AI 把价格打到成本线以下,一晚上损失几万。价格调整可以 AI 建议、人审核,不能全自动
>
> **Review 反哺选品要看统计显著性**——一条'包装差'差评不值得投 10 万改包装,要看 10% 以上客户是否普遍反馈。"

---

### 9-6 ⑥ 履约 / 物流(你的主战场)

**AI 擅长点**:智能优选渠道 / 物流异常预警 / 拦截判断 / 到货时效预测 / 退货包裹识别

**3 大坑点 + 解法**:
1. **物流数据质量参差** → 标准化数据字典+多数据源交叉验证
2. **自动拦截误伤真实订单** → 分档拦截策略(查无/超时/永久失败3档)
3. **时效预测依赖季节性** → 加旺季/平季特征+人工兜底提示

**面试话术**:(参考你 Y2 下 call 7 档故事,这块你最熟不用背)

---

### 9-7 ⑦ 售后 / 数据复盘

**AI 擅长点**:差评申诉话术 / Review 自动回复 / 月度利润核算 / 异常订单识别

**2 大坑点 + 解法**:
1. **差评回复话术太套路** → AI 生成初稿+客服个性化修改
2. **利润核算数据源打通难** → 先做 MVP(订单+广告 70%准确度)再补全

### 9-8 AI 赋能通用方法论(终极大招话术)

> "讲了 7 个环节的细节,**我想收一下,总结一个通用方法论**——
>
> AI 能做什么不能做什么,有 3 个判断维度:
>
> **数据密集度**:AI 擅长的都是数据密集场景,因为 LLM 本质是数据压缩器
>
> **容错成本**:容错成本低的(AI 生图不满意可重画)可以大胆自动化;容错成本高的(自动改价/自动下单)必须人工兜底
>
> **创造性要求**:低创造性(数据筛选、格式转换)AI 完全胜任;高创造性(品牌战略、市场节奏判断)AI 只能辅助
>
> 所以我做 AI 产品的心智模型是:**把复杂业务拆到最小决策单元,再判断每个单元适不适合 AI——适合的全自动化,不适合的做 AI 建议+人工 Review**。这个方法论在 OpenClaw 和 Echo 都验证过,也是我来贵司希望能带来的价值。"

### ① 选品 / 开发

### ④ 美工 / 视觉资产

### ⑤ 广告 / 运营

- RPA 价值大：**批量上下架、批量改价、批量报名活动**（影刀 / n8n 都能做）

### ⑥ 履约 / 物流（自己主战场）

### ⑦ 售后 / 数据复盘

## 十一、业务认知追问预演（15 道高频题）

### AI Listing 方向
4. **"用 AI 写 Listing 会不会被平台降权？怎么避免？"**

### 高频题答题锚点（速记）

**Q1-3 选品**：
> 核心是**"缩短人工决策信息处理成本"**；数据维度 = 销量趋势 + 价格带 + 竞品数量 + Review 反馈；侵权必须**前置过滤**（商标库 + 专利库）。

**Q4-6 Listing**：
> 三大坑 = **堆砌降权 / 调性不一致 / 禁用词**；解法 = **品牌调性 + 关键词库 + 合规库三模板**；长尾词要**自然融入**不是拼接。

**Q7-9 AI 生图**：
> 主图严格白底合规（**慎用纯 AI**），二图场景化可用；**LoRA 保品牌一致**；批量生图**500 张里人工筛 50 张**才是正确姿势。

**Q10-11 AI Agent**：
> 准确率靠**拆解最小单元**（AI 判断 + CLI 执行）；错单靠**人工 review 关卡 + 一键回滚**；5 层架构是**职责解耦 + 每层可独立迭代**。

**Q12-13 工作流 / RPA**：
> n8n 适合**自定义程度高、技术深度深**；coze 适合**LLM 能力强、开箱即用**；**高频 + 重复 + 规则明确 + 低容错**的适合 RPA。

**Q14-15 产品方法论**：
> B 端 AI **容错和兜底是生死线**（一次错误几十万损失）；**MVP 定义 = 端到端打通一个高频场景**，不是"做多个半成品"；**成功度量 = 用户日活 + 单次使用节省时长 + 业务 ROI 三个指标**。

## 十二、面试心态 & 沟通原则

### 沟通风格
- **结构化**：每个回答先给结论，再展开 1-2 个例子
- **不背稿感**：多用"举个例子"、"比如"、"我之前做的时候"
- **主动抛钩**："这里我有个有意思的细节可以展开讲一下"

### 数字可以大胆讲的
- **马帮 1800+ 物流商、1000+ 海外仓**（真实）
- **月处理订单月均亿级**（保密已脱敏）
- **OpenClaw 日均 400+ 单异常处理** + **Echo 30 人使用 / 单周 1000+ 对话**（真实）
- **2 款企业级 AI 产品独立全栈开发**（真实）

### 数字要谨慎的（避开精确数字 + 客户名）
- ❌ "完美日记 / 逸仙电商 / 单项目 140 万" → ✅ "头部美妆出海集团 / 单项目百万级"
- ❌ "年营收 300 万" → ✅ "数百万级"
- ❌ "663 家海外仓、同比 +60%" → ✅ "数百家、同比约 70%"

### 绝对不要讲
- ❌ 马帮任何客户具体名字
- ❌ 任何内部 API 参数、权重数值
- ❌ 任何员工名字（即使是公开的 Ruben 也用"核心卖家"代称）
- ❌ 任何具体合作物流商的商务价格
