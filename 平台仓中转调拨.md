# 平台仓中转调拨 - 物流中台新业务场景

## 业务背景

跨境卖家将货物从国内发往海外仓，再从海外仓中转调拨到平台仓（如美客多官方仓）。由于平台仓库容限制，无法全部走国内直达平台仓的方式，必须经过海外仓中转。

中转调拨分两种业务模式：

### 原箱中转
- 国内已提前贴好平台仓商品标签
- 货物到海外仓后**无需拆箱**
- 客户只需通过 ERP 下发箱唛给海外仓即可

### 拆箱换标（更复杂）
- 国内未贴平台仓商品标签
- 货物到海外仓后，仓库需要**拆箱 → 贴标 → 重新装箱**
- 仓库完成装箱后，将装箱数据回传 ERP
- 客户基于装箱数据从平台（美客多）获取箱唛和入库预约信
- 再通过 ERP 将箱唛 + 预约信下发给海外仓

## 核心产品设计

### 问题根源
运营人员从美客多商家中心下载**商品标签、箱唛、预约信**三类附件，上传到 ERP 后统一下发给海外仓。但原有系统无法区分附件类型，仓库收到后无法判断哪个是商品标签、哪个是箱唛，导致入库效率低甚至被拒收。

### 解决方案
对出库单附件做**分类传输**改造：

**数据结构调整（`ship_batch_order` 表新增字段）：**
- `transfer_biz_type`：业务类型（1: 拆箱换标 / 2: 原箱中转）
- `transfer_process_status`：流程状态（10: 初始 / 20: 已下发换标任务 / 30: 装箱完成）

**拆箱换标流程（b-2-3）：**
1. 客户上传商品标签，ERP 下发换标任务给海外仓（`transfer_process_status=20`，下发按钮置灰，防止重复下发）
2. 海外仓完成拆箱贴标装箱，回传装箱数据到 ERP（`transfer_process_status=30`）
3. 客户基于装箱数据从美客多获取箱唛 + 预约信，上传 ERP
4. ERP 二次下发箱唛 + 预约信给海外仓（按钮重新高亮可用）

**原箱中转流程：**
1. 客户直接上传箱唛给 ERP
2. ERP 一次性下发箱唛给海外仓

**按钮状态控制逻辑：**
- `logsiticsId=3814` 且 `transfer_process_status=20`：按钮置灰，悬浮提示「当前已下发换标任务，请等待海外仓装箱完毕」
- `logsiticsId=3814` 且 `transfer_process_status=30` 且 `order_status=1`：按钮高亮，可进行二次下发
