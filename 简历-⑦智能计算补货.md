# ⑦ 智能计算补货

**海外仓智能补货推荐 · 多策略日销量预测 · 补货量自动计算**
时间：2024–2025 | 角色：产品负责人 + 开发参与

> 马帮海外仓备货域核心能力，解决卖家"补多少、什么时候补"的决策问题，从人工经验判断升级为系统自动推荐。

---

## 一、背景与痛点

跨境卖家在海外仓备货时面临两个核心问题：
- **补多少**：补少了断货损失销量，补多了积压占用资金
- **什么时候补**：补晚了来不及，补早了库存周转率低

原来全靠运营人员人工经验判断，日均管理数百个SKU时极易出错，断货率和积压率居高不下。

---

## 二、产品设计

### 2.1 核心补货公式

```
补货量 = (入仓可售天数 + 物流时效 + 入仓操作天数) × 预测日销量
        + 待发货量（订单/调拨/手工出库）
        - 现有库存（实际库存 + 各类在途量）
        + 自定义数量
```

**参数说明**：
- **入仓可售天数**：补货到货后希望能卖多少天（客户自配）
- **物流时效**：从下补货单到货到仓的运输天数
- **入仓操作天数**：到仓后入库操作所需天数
- **预测日销量**：系统按策略计算（见2.2）
- **在途量**：采购在途 + 调拨在途 + 海外仓调拨在途 + 加工在途等多维度

### 2.2 日销量预测四策略

系统支持四种日销量计算策略，客户按SKU/时间段灵活配置：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **固定销量** | 直接设置固定日销量值 | 新品/促销期/销量稳定品 |
| **动态加权销量** | 7/14/28/42天历史日均销量加权平均 | 常规品，自动跟随销量趋势 |
| **历史区间预测** | 指定历史时间段算日均，乘以系数推算未来 | 季节性商品、参考去年同期 |
| **历史修正** | 对异常历史数据去噪（固定值替换或百分比修正） | 历史有促销/断货导致数据失真 |

**动态加权公式**：
```
预测日销量 = (近7天日均 × W7 + 近14天日均 × W14 + 近28天日均 × W28 + 近42天日均 × W42) / 总权重
```
权重由客户自定义，默认各25%均等权重。

### 2.3 库存状态判断

系统自动判断每个SKU的库存健康状态：

```
可售天数 = 可用库存 ÷ 预测日销量

安全：可售天数 - 物流时效 - 入仓操作天数 > 警戒天数
预警：0 < 可售天数 - 物流时效 - 入仓操作天数 ≤ 警戒天数
断货：可售天数 - 物流时效 - 入仓操作天数 ≤ 0
```

同时输出：**预计断货时间**、**建议补货日期**（断货时间与警戒线取早者）。

### 2.4 备货周期逐日模拟

系统对每个SKU做备货周期内的逐日销量模拟：
- 按天扣减可用库存，判断哪天会断货
- 到货入库时间点自动补充库存
- 输出每天的库存状态和销量明细，让卖家看清楚"为什么建议补这么多"

### 2.5 多物流方案并行计算

同一SKU支持配置多条物流线路（如空运/海运），系统对每条线路独立计算补货建议，卖家可对比选择。

---

## 三、技术实现要点

- **异步并发**：按仓库维度并发计算，多仓同时跑互不阻塞（CompletableFuture线程池）
- **增量触发**：支持全量定时计算（每日Job）+ 单品实时触发（MQ消息）两种模式
- **公式引擎**：补货量计算公式通过表达式引擎动态执行，客户可自定义公式变量组合
- **过滤逻辑**：42天无销量的SKU自动跳过计算，避免无效计算

---

## 四、结果

- 覆盖海外仓备货场景，支持客户按SKU/仓库维度灵活配置补货策略
- 系统自动输出补货建议量、建议补货日期、预计断货时间，运营从"人工算"变成"看推荐确认"
- 规划逐步覆盖采购团队日均数千SKU的补货决策自动化
